# Task 4.1 — Identify Service Boundaries

## Objective

Determine how the monolithic Coffee Suppliers application should be split into separate microservices.

## Actions Performed

- Reviewed the existing monolithic application structure
- Identified which routes were used by customers and which were used by administrators
- Grouped supplier viewing functionality into a customer-facing service
- Grouped supplier management functionality into an employee/admin service

## Result

Clear service boundaries were defined, allowing the monolithic application to be split into two microservices in the following tasks.
